GA4 : 5 erreurs de configuration qui faussent vos données (et comment les réparer)

De nombreuses entreprises pensent être "bien configurées" simplement parce qu'elles voient des chiffres monter dans leurs rapports GA4. Pourtant, j'audite chaque semaine des comptes où 30% à 50% de la donnée est soit manquante, soit totalement erronée. Une mauvaise donnée est plus dangereuse que pas de donnée du tout.

Sommaire 1. La rétention des données limitée à 2 mois 2. L'absence d'exclusion des sites référents 3. Confondre Événements et Conversions 4. Ignorer le Consent Mode V2 et le RGPD 5. Ne pas tester via le DebugView

1. La rétention des données limitée à 2 mois

C'est l'erreur "silencieuse" par excellence. Par défaut, GA4 configure la rétention des données d'exploration sur 2 mois seulement. Si vous voulez comparer vos performances actuelles avec celles de l'année dernière dans l'onglet "Explorer", vous n'aurez aucun historique. Pour un business saisonnier, c'est une catastrophe stratégique.

La solution : Administration → Paramètres des données → Conservation des données. Passez de "2 mois" à "14 mois" et enregistrez immédiatement.

2. L'absence d'exclusion des sites référents

Votre canal "Referral" contient souvent votre propre nom de domaine ou des plateformes de paiement comme Stripe ou PayPal. Lorsqu'un client quitte votre site pour payer et revient après validation, GA4 peut considérer cela comme une nouvelle session. Résultat : la vente est attribuée à "paypal.com" au lieu de votre campagne SEO ou Google Ads. Votre analyse de ROI est alors totalement faussée.

3. Confondre Événements et Conversions

Dans GA4, "tout est un événement". Un clic, un scroll, une vue de page. Mais tout n'est pas une conversion. L'erreur courante : marquer trop d'événements comme conversions. Si votre taux de conversion affiche 80%, vous ne mesurez plus votre business, vous mesurez le bruit. Une conversion doit avoir une valeur économique directe : formulaire rempli, achat, demande de devis.

Conséquence : Si vous liez un compte GA4 mal configuré à Google Ads, l'IA de Google va optimiser vos publicités pour des actions inutiles, gaspillant ainsi votre budget.

4. Ignorer le Consent Mode V2 et le RGPD

Depuis mars 2024, le Consent Mode V2 est obligatoire pour continuer à utiliser les fonctionnalités de ciblage de Google. Sans une bonne liaison entre votre bannière de cookies (CMP) et GA4, vous perdez une quantité massive de données pour les utilisateurs refusant les cookies. Le Consent Mode V2 permet la "modélisation de données" pour estimer les conversions perdues, tout en restant 100% conforme au RGPD.

5. Ne pas tester via le DebugView

Beaucoup installent des tags via GTM et vérifient simplement si les chiffres montent dans le rapport "Temps réel". C'est insuffisant. Le DebugView dans GA4 est la seule manière de vérifier que les paramètres envoyés sont corrects : valeur de transaction bien formatée, devise correcte, ID de transaction unique. Sans test rigoureux, vous accumulez des données "sales" (Garbage In, Garbage Out).
